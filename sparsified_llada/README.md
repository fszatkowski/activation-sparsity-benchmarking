# Sparsified LLaDA Evaluations

This repository provides a comprehensive framework for evaluating the LLaDA under various activation sparsity constraints. It is designed to run efficiently on HPC clusters using SLURM, supporting multi-GPU distributed evaluation, automated parameter sweeps, and detailed analysis of mask stability and sparsity dynamics across diffusion steps.

## 🚀 Getting Started: Running Evaluations
The evaluation process is automated via SLURM bash scripts. These scripts manage the environment setup, dynamic configuration of the accelerate library, and job submission.


### 1. Environment Setup
Before running the scripts, ensure your Python environment is prepared as discussed in the setup instructions:

```bash
cd sparsified_llada

python3 -m venv venv_llada

source venv_llada/bin/activate

pip install -e .
```

### 2. Configuration & Submission

There are two primary submission scripts depending on which layers you wish to sparsify:

* `llada_input_intermediate.sh`: Applies sparsity hooks to both input (gate/up projection) and intermediate (down projection) layers.
* `llada_intermediate.sh`: Applies sparsity hooks only to the intermediate (down projection) layers.

in both of them modify

* line `source <path to python interpreter>` in line `88`.
* add variable `HARNESS_DIR` and `LLADA_DIR`.

#### How to run:

```bash
export SLURM_ACC="your_account"
export SLURM_PARTITION="gpu_partition"
export VENV_PATH="/path/to/venv/bin/activate"
export BASE_DIR="/path/to/project_root"
export HF_TOKEN_VAL="your_huggingface_token"

./llada_input_intermediate.sh
```

### 3. What these scripts do

1. Validation: Asserts all required environment variables and paths are set.
2. Task Definition: Loops through a predefined set of tasks (e.g., humaneval, mbpp, arc_challenge, mmlu, etc.).
3. Parameter Sweeping: For each task, it iterates through a list of sparsity parameters (top-p thresholds).
4. Dynamic Acceleration: For every specific job, it generates a unique .accelerate_config.yaml to ensure correct multi-GPU mapping (num_processes, gpu_ids).
5. Job Orchestration: Submits a temporary sbatch script that loads necessary modules (CUDA, ML-bundles), activates the environment, and executes the evaluation script.

## 📂 Job Outputs & Data Artifacts

Each SLURM job creates a dedicated results directory under WORKDIR. Upon completion, the following files are generated by the evaluation scripts (llada_top_p_at_diff_int.py and llada_top_p_at_diff_inp_int.py):

| File Suffix | Format | Description |
| :--- | :--- | :--- |
| `_summary.csv` | CSV | Layer-wise and aggregated Jaccard indices (Consecutive and Drift). |
| `_per_step.json/csv` | JSON/CSV | Detailed Jaccard metrics broken down by every individual diffusion step. |
| `_tokenaware_per_step.json` | JSON | Aggregated Jaccard metrics calculated per-token rather than globally. |
| `_sparsity_per_step.json/csv` | JSON/CSV | Records the actual mean sparsity percentage achieved at each diffusion step per layer. |
| `_mask_snapshots.npz` | NumPy | Compressed binary containing raw mask indices and magnitudes for specific steps (used for stability analysis). |
| `_neuron_correlation.json` | JSON | Analysis correlating initial activation magnitudes with final mask frequency. |
